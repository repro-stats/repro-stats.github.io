Packages

Five openly developed packages — two on CRAN today, three in active development.

reproducr CRAN

The reproducibility foundation. Audit, certify, and monitor R scripts and environments for regulated settings.

  • Script-level audit trail with cryptographic hash
  • Risk scoring and environment certification
  • Lockfiles and drift detection
install.packages("reproducr")

Docs & source →

bayprior CRAN Shiny app

Structured Bayesian prior elicitation, conflict diagnostics, sensitivity analysis, and regulatory reporting. Aligned with FDA 2026 draft guidance.

  • SHELF-style elicitation across 6 distribution families
  • Prior-data conflict diagnostics
  • Regulatory prior justification reports (HTML/PDF/Word)
install.packages("bayprior")

Docs & source →

regulog In development

Electronic audit logging for R, designed around 21 CFR Part 11 requirements.

  • Immutable audit log with digital signatures
  • User-attributed action tracking
  • Deep integration with reproducr
remotes::install_github("repro-stats/regulog")

View on GitHub →

lineager In development

Row-level data provenance for clinical datasets. Track every transformation from SDTM to ADaM.

  • Row-level transformation tracking
  • SDTM to ADaM derivation audit trail
  • CDISC-compatible lineage documentation
remotes::install_github("repro-stats/lineager")

View on GitHub →

estimandr In development

ICH E9(R1) estimand framework for R. Define, document, and implement your estimand strategy directly in your analysis pipeline. Designed to pair with bayprior for Bayesian estimand analyses.

  • Structured estimand specification objects
  • Intercurrent event strategy documentation
  • SAP-ready estimand output tables
remotes::install_github("repro-stats/estimandr")

About

Reproducibility in clinical analyses is harder than it should be. R environments go undocumented, package versions drift, and the steps between raw data and submission output are rarely captured in a way that supports independent verification.

ReproStats is an open-source project building the R infrastructure to address this. Five packages covering reproducibility and certification (reproducr, regulog), Bayesian prior methodology (bayprior), data provenance (lineager), and estimand specification (estimandr).

reproducr and bayprior are on CRAN. The remaining three are in active development. All packages are openly developed on GitHub and designed for GxP environments.

ReproStats is maintained by Ndoh Penn, a biostatistician based in Brussels, Belgium. The enterprise platform built on this foundation is Reprosia.

Questions, contributions, or collaboration → open an issue on GitHub or penn@reprosia.com.
